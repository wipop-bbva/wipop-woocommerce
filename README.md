# Wipöp WooCommerce Plugin

WooCommerce plugin that integrates the Wipöp payment gateway, allowing your WordPress e-commerce to accept payments easily.  
It includes production-ready gateways for card and Bizum. Configure shared credentials across all methods and keep WooCommerce orders in sync with the transactions reported by Wipöp.

## Features

- **Card payments**
  - Redirect checkout experience using Wipöp payment links.
  - Integrated COF tokenization to store cards and enable one-click charges or custom recurring payments.
  - Preauthorization support with order actions to capture or void.
  - Full and partial refunds from the WooCommerce order screen.
- **Bizum payments**
  - Dedicated button at checkout that creates a Bizum charge.
- **Google Pay**
  - Initial support pending.
- **Merchant synchronization and admin tools**
  - Central settings page at `WooCommerce > Wipop` with environment, keys, terminal, and capture mode.
  - Credential verification that saves allowed methods.
  - “Recurring payment” tab in products plus the WP-Cron that schedules monthly/yearly MIT charges reusing stored COF tokens.
- **Order status updates**
  - Webhook endpoint `https://your-store.com/?wc-api=wipop_bbva` that updates orders on confirmation, failure, refund, capture, and reversal events.
  - Logging via the WooCommerce logger to audit each action.

## Requirements

- PHP 8.1 or higher.
- WordPress with WooCommerce installed and active.
- Wipöp merchant account with Sandbox and Production credentials (Merchant ID, Terminal ID, Public Key, Private Key).

## Installation

### Install from ZIP (stores)

1. Download the `wipop.zip` package generated by the pipeline or your build system.
2. In WordPress go to `Plugins → Add New → Upload Plugin`, select the ZIP and click **Install Now**.
3. Activate the plugin and continue with configuration.

### Local development

- Follow the development guide in [development.md](development.md).

## Before you start

- Complete onboarding with Wipöp to get Sandbox and Production terminals.
- Gather all plugin credentials: Merchant ID, Terminal ID, Public Key and Private Key.
- Ensure the store is reachable via HTTPS.
- Configure `https://{your-domain}/?wc-api=wipop_bbva` as the webhook URL in the Wipöp/BBVA portal.
- Copy the webhook username/password generated in `WooCommerce > Wipop` into the portal.

## Quick start

1. Go to `WooCommerce > Wipop` and fill in Merchant ID, Terminal, public/private keys and the target environment.
2. Choose the **Preauthorizations** mode (`Capture automatically` or `Preauthorize only`) for card payments.
3. Click **Verify data** to validate credentials and fetch allowed payment methods.
4. Go to `WooCommerce > Payments` and enable the gateways you want to show (Card, Bizum, Google Pay).
5. Place a test order in Sandbox to verify the flow. Make sure Wipöp metadata is stored and the webhook updates the status.
6. Optional: mark a product as recurring (Edit product → **Recurring payment** tab) so that after the first charge, the plugin schedules automatic MIT charges according to the selected frequency.

## Configuration

### Credentials and environment

Settings are stored in the `wipop_settings` option. Required fields:

- `Merchant ID`: as provided in Wipöp.
- `Public Key` / `Private Key`: cryptographic keys to sign requests.
- `Terminal ID`: number between 0 and 99.
- `Environment`: Sandbox or Production.

### Manual capture / preauthorization

Select `Preauthorize only` in **Preauthorizations** if you want card charges to be authorized but left pending capture. When enabled:

- The order goes to `on-hold` with a note indicating it must be captured.
- The actions *Capture preauthorization with Wipöp* and *Void preauthorization with Wipöp* appear in the order actions dropdown.
- Preauthorizations typically have a validity period of about one week.

With `Capture automatically` (default behavior) the capture is confirmed at the time of the charge.

### Payment methods

The plugin queries available payment methods and only registers the gateways returned by Wipöp. Enable or disable each method from `WooCommerce > Payments`.

### Webhooks

Expose `https://{your-domain}/?wc-api=wipop_bbva` in the BBVA portal with the credentials shown in `WooCommerce > Wipop`. The webhook syncs:

- Order status changes (pending → on-hold → processing/completed → failed).
- Manual capture metadata.
- Transaction identifiers, masked card data and stored tokens.
- Refund events.
- Verification events to mark the webhook as connected in the admin panel.

### Recurring products

When editing products you’ll see the **Recurring payment** tab. Enable it and choose `Monthly` or `Yearly` so items are flagged as recurring. Once the first charge is marked as paid, the plugin:

- Verifies that the customer has a stored token.
- Groups amounts by frequency and stores order metadata with the amount, next date, and execution counters.
- Schedules the job via WP-Cron.
- Adds order notes for each attempt and stops recurrence if the order becomes inactive, card/token data is missing, amount is invalid, or a recurring charge fails.

Recurring charges can also be stopped manually from the order actions dropdown with **Stop recurring charges with Wipöp**.

Recurring charges are automatically disabled if the order becomes cancelled/refunded/failed, if the order is deleted, or if the customer deletes the card (no COF token found).

## Error handling and logs

- API failures display admin notices.
- Detailed logs are available under `WooCommerce > Status > Logs`.
